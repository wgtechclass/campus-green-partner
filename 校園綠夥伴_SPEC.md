# SPEC｜《校園綠夥伴：環境復甦任務》遊戲化課程系統需求規格書

## 1. 專案概述

### 1.1 專案名稱

《校園綠夥伴：環境復甦任務》

### 1.2 專案目的

本系統是一款結合真實校園行動與虛擬寵物養成的網頁式任務遊戲，適用於國小高年級四週課程。

學生以小組為單位，透過校園探索、環境觀察、垃圾熱點紀錄、原因分析與改善提案，累積綠夥伴的能力值，解鎖徽章與進化階段，最後產出小組的「校園綠夥伴成果卡」。

### 1.3 課程定位

本系統可支援下列課程類型：

- 校訂彈性學習課程
- 綜合活動課程
- 環境教育主題課程
- 健康促進與校園行動課程
- 國小高年級跨域探究課程

### 1.4 適用對象

| 對象 | 說明 |
|---|---|
| 學生 | 國小五、六年級學生 |
| 教師 | 課程設計者、導師、綜合活動教師或行政推動者 |
| 使用單位 | 班級、小組、年級或全校高年級 |

---

## 2. 核心玩法

### 2.1 遊戲核心循環

學生在四週內依序完成任務：

> 接任務 → 走校園 → 完成觀察 → 回報資料 → 獲得能力值 → 綠夥伴成長 → 解鎖下一階段 → 產生成果卡

### 2.2 玩家單位

本遊戲採用小組制。

| 項目 | 設定 |
|---|---|
| 每組人數 | 4～5 人 |
| 每組單位 | 一支校園調查小隊 |
| 每組角色 | 共同養成一隻綠夥伴 |
| 學生登入方式 | 選擇班級、組別、輸入隊名 |
| 不採用 | 個人帳號、密碼、GPS 定位 |

---

## 3. 遊戲世界觀

校園裡住著一群「綠夥伴」，牠們依靠乾淨的校園環境與學生的行動能量成長。

最近校園中出現許多「髒亂能量點」，讓綠夥伴逐漸失去力量。學生需要組成小隊，走入校園完成偵查、淨化、分析與倡議任務，幫助綠夥伴恢復能量，並讓校園重新變得乾淨、有活力。

---

## 4. 四週任務架構

| 週次 | 關卡名稱 | 課程重點 | 遊戲任務 | 主要產出 |
|---|---|---|---|---|
| 第 1 週 | 綠夥伴甦醒 | 建立任務感與小隊認同 | 建立小隊、選擇綠夥伴、預測校園熱點 | 小隊資料、綠夥伴設定 |
| 第 2 週 | 校園偵查任務 | 走校園與環境觀察 | 到指定區域觀察與回報 | 校園觀察紀錄 |
| 第 3 週 | 熱點分析任務 | 分析問題與提出改善 | 選定熱點、分析原因、提出做法 | 改善提案草稿 |
| 第 4 週 | 校園守護任務 | 成果表達與全校倡議 | 完成宣導作品與成果卡 | 成果卡、宣導口號、提案 |

## 5. 綠夥伴角色系統

### 5.1 初始角色

系統提供數種初始綠夥伴供學生選擇。

| 綠夥伴 | 個性 | 初始特色 |
|---|---|---|
| 小葉靈 | 溫和、愛觀察 | 偵查值較高 |
| 步步獸 | 活潑、愛探索 | 體力值較高 |
| 淨淨鳥 | 喜歡乾淨 | 淨化值較高 |
| 亮光鹿 | 會發現問題 | 智慧值較高 |
| 花芽龍 | 成長速度快 | 影響力較高 |

### 5.2 進化階段

| 階段 | 對應週次 | 條件 |
|---|---|---|
| 種子期 | 第 1 週 | 建立小隊與選擇綠夥伴 |
| 幼體期 | 第 2 週 | 完成至少一筆校園偵查回報 |
| 成長期 | 第 3 週 | 完成熱點分析與改善提案 |
| 守護期 | 第 4 週 | 完成宣導作品與成果卡 |

## 6. 能力值系統

### 6.1 能力值類型

| 能力值 | 代表意義 | 取得方式 |
|---|---|---|
| 體力值 | 願意走出教室、完成校園探索 | 完成校園走動任務 |
| 偵查值 | 能看見環境問題 | 提交有效觀察紀錄 |
| 淨化值 | 願意採取環境行動 | 完成清潔、改善或環境行動 |
| 智慧值 | 能分析原因與分類問題 | 說明垃圾類型與可能原因 |
| 影響力 | 能讓更多人注意問題 | 完成宣導作品、口號或提案 |

### 6.2 得分規則

| 行動 | 能力值增加 |
|---|---:|
| 建立小隊 | 體力值 +5 |
| 選擇並命名綠夥伴 | 影響力 +5 |
| 完成一筆校園觀察紀錄 | 偵查值 +10 |
| 填寫垃圾類型 | 智慧值 +5 |
| 說明可能原因 | 智慧值 +10 |
| 提出改善想法 | 淨化值 +10 |
| 完成熱點分析任務 | 智慧值 +20 |
| 完成宣導口號 | 影響力 +10 |
| 完成成果卡 | 影響力 +20、淨化值 +10 |

## 7. 徽章系統

| 徽章名稱 | 解鎖條件 |
|---|---|
| 甦醒徽章 | 完成小隊建立與綠夥伴命名 |
| 校園偵查徽章 | 完成至少一筆校園觀察紀錄 |
| 熱點分析徽章 | 完成一個垃圾熱點分析 |
| 改善提案徽章 | 提出具體改善方法 |
| 校園守護徽章 | 完成最終成果卡 |
| 影響力徽章 | 完成宣導口號、廣播稿或海報文案 |

## 8. 系統角色

### 8.1 學生端

學生端主要功能：

1. 建立小隊
2. 選擇綠夥伴
3. 查看任務地圖
4. 填寫任務回報
5. 查看綠夥伴成長狀態
6. 查看已解鎖徽章
7. 產生小組成果卡

教師端主要功能：

1. 查看所有小隊資料
2. 查看各組任務進度
3. 查看學生觀察紀錄
4. 查看校園垃圾熱點統計
5. 查看各組能力值
6. 查看各組改善提案
7. 產生全班或全年級成果摘要
8. 管理任務開放狀態

## 13. AI 介入設計

### 13.1 MVP 階段

MVP 階段不直接串接 AI API，避免開發與權限複雜化。

系統提供：

1. 教師可複製學生回報資料
2. 系統產生整理用 prompt
3. 教師可將資料貼到 ChatGPT 分析
4. 分析結果再貼回教師端成果摘要欄位

### 13.2 AI 可協助內容

| AI 任務 | 說明 |
|---|---|
| 資料整理 | 整理常見垃圾熱點、垃圾類型與可能原因 |
| 提案回饋 | 對學生改善方法提出追問與建議 |
| 宣導文案 | 協助產生口號、廣播稿、海報文案 |
| 成果摘要 | 協助教師產生全班或全年級成果報告 |
| 綠夥伴回饋 | 依據小組任務表現生成鼓勵文字 |

## 14. 成果卡需求

每組遊戲結束後，系統應產生一張「校園綠夥伴成果卡」。

成果卡內容包含：

| 欄位 | 說明 |
|---|---|
| 班級 | 小組所屬班級 |
| 組別 | 小組編號 |
| 隊名 | 小組隊名 |
| 綠夥伴名稱 | 學生命名 |
| 綠夥伴最終型態 | 種子期、幼體期、成長期或守護期 |
| 已獲得徽章 | 顯示徽章列表 |
| 調查地點 | 小組曾調查的地點 |
| 主要發現 | 小組最重要的環境發現 |
| 可能原因 | 小組分析結果 |
| 改善提案 | 小組提出的具體方法 |
| 宣導口號 | 小組產生的倡議語 |
| 行動承諾 | 小組未來願意持續做到的行動 |
